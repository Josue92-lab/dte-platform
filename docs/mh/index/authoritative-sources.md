# Authoritative Sources Classification

Not all documents provided by the MH dictate software behavior. This classification separates binding technical requirements from administrative guidance.

### AUTHORITATIVE
*These documents dictate exact software behavior, rules, formats, and API contracts. They strictly govern implementation decisions.*
- `Normativa de Cumplimiento de los Documentos Tributarios Electrónicos_Versión 2.0.pdf`
- `Catálogos - Facturación Electrónica.xlsx` (and `.pdf` equivalent)
- `Manual Técnico para la Integración Tecnológica del Sistema de Transmisión v2.pdf`
- `Manual Funcional del Sistema de Transmisión V 2.0.pdf`
- `svfe-json-schemas/` (All contained JSON schemas)

### REFERENCE
*These documents provide administrative context, onboarding requirements, or legal background, but do not dictate code.*
- `Aviso mandatario referente a ventas con factura.pdf`
- `Manual de acreditamiento y obtención de certificado de firma electrónica.pdf`

### EXAMPLE
*These are reference implementations provided to assist development, but our platform must implement its own equivalent.*
- `svfe-api-firmador/` (Java reference implementation)

### HISTORICAL / ADMINISTRATIVE
*User manuals and onboarding checklists for operational staff. Irrelevant to code.*
- `Guía del Proceso de Incorporación para ser Emisor de Documentos Tributarios Electrónicos.pdf`
- `Manual de Usuario de Consulta Pública de los DTE y Eventos.pdf`
- `Manual de Usuario del Sitio de Emisores DTE.pdf`
- `Manual del Usuario para la Solicitud de Ingreso al ambiente para pruebas.pdf`
- `Manual para la obtención de la Autorización como Emisor de DTE.pdf`
